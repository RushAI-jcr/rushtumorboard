## Problem Statement

`_fallback_doc_content` in `content_export.py` was described in the PR as setting `action_items=["⚠ Export used LLM fallback — review all fields before printing."]` to warn clinicians that the document was generated by a degraded fallback path. However, the committed code had `action_items=[]` — the warning was absent. Clinicians would receive a Word document with blank Col 0 fields (MRN, attending initials, RTC, main location, path date) with no visual indicator that these fields required verification before printing at a live tumor board.

The Col 4 `_build_col4_richtext` renderer renders `action_items` in RED in the Word document — exactly the right place for this warning to appear to the clinician.

## Findings

- **File:** `src/scenarios/default/tools/content_export/content_export.py`, line 455
- **Reported by:** agent-native-reviewer
- **Severity:** P1 — clinical safety issue; clinicians print documents with blank critical fields

## Resolution

Fixed inline during code review (2026-04-02):

```python
# Before:
action_items=[],

# After:
action_items=["⚠ Export used LLM fallback — review all fields before printing."],
```
